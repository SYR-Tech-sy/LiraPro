قم الآن بربط الموقع بالكامل مع قاعدة بيانات Supabase بشكل احترافي ومنظم.

المطلوب تنفيذ التالي بالكامل:

1- إنشاء وربط جميع جداول قاعدة البيانات داخل Supabase وربطها بالموقع ولوحة تحكم الأدمن.

2- استخدام معرف UID / UUID فريد لكل:

- مستخدم
- مزود أسعار
- شركة
- إشعار
- طلب توثيق
- طلب حذف
- جلسة
- سعر
- سلعة
- فئة
- عملية إدارية

3- حفظ وربط جميع بيانات الموقع داخل Supabase بشكل كامل، بما يشمل:

- بيانات المستخدمين
- بيانات مزودي الأسعار
- بيانات الشركات
- الجلسات والأجهزة
- الإشعارات
- الأسعار
- العملات والمعادن
- الطلبات
- الحظر والتقييد
- التوثيق
- السجلات الإدارية
- النشاطات داخل الموقع

4- ربط جميع الصفحات داخل لوحة تحكم الأدمن بقاعدة البيانات مباشرة بحيث:

- تعرض البيانات الحقيقية مباشرة من Supabase.
- يتم تحديث البيانات لحظياً.
- يتم حفظ أي تعديل مباشرة داخل قاعدة البيانات.

5- في صفحة المستخدمين داخل لوحة الأدمن:
اعرض:

- UID / UUID
- جميع البيانات الشخصية
- الجلسات والأجهزة
- نوع تسجيل الدخول
- الموقع الجغرافي
- حالة الحساب
- الإشعارات
- النشاطات

6- في صفحة التجار:
اعرض:

- UID الخاص بالتاجر
- جميع بياناته
- الأسعار التي أضافها
- الفئات التابعة له
- المدن والمحافظات
- حالة الحساب
- سجل النشاطات

7- في صفحة الطلبات:
اربط:

- طلبات التوثيق
- طلبات الحذف
- طلبات مزودي الأسعار

مع UID الخاص بكل مستخدم واعرض البيانات كاملة.

8- قم بإنشاء جداول منفصلة ومنظمة داخل Supabase لكل قسم وفئة بالموقع، مع تسمية واضحة واحترافية للجداول.

مثال:

- users
- user_sessions
- user_notifications
- user_devices
- merchants
- merchant_prices
- merchant_categories
- merchant_requests
- currencies
- metals
- categories
- products
- notifications
- admin_logs
- verification_requests
- delete_requests
- restricted_accounts
- banned_accounts
- reports
- app_settings

وأنشئ أي جداول إضافية مطلوبة تلقائياً حسب بنية الموقع.

9- تأكد من:

- إنشاء العلاقات بين الجداول بشكل صحيح.
- استخدام Foreign Keys.
- منع تكرار البيانات.
- مزامنة البيانات بشكل لحظي Real-time.
- حماية البيانات باستخدام صلاحيات Supabase و RLS Policies.

10- قم بإنشاء جميع الجداول الناقصة تلقائياً إذا لم تكن موجودة.

11- اربط جميع الأيقونات والصفحات داخل لوحة تحكم الأدمن ببيانات Supabase الحقيقية بدلاً من البيانات الوهمية أو المحلية.

12- تأكد من أن:

- أي مستخدم جديد يظهر فوراً داخل لوحة الأدمن.
- أي تعديل يظهر مباشرة.
- أي إشعار أو إجراء إداري يتم حفظه وربطه بمعرف المستخدم الخاص به.

13- نظف النظام من أي:

- بيانات تجريبية
- Local Storage غير ضروري
- بيانات وهمية Mock Data
- APIs قديمة غير مستخدمة

واجعل Supabase هو المصدر الأساسي والوحيد لجميع بيانات الموقع.
